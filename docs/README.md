# Project Documentation

Этот каталог - рабочая память проекта BootCamp bots. Сюда фиксируем архитектуру, деплой, проверки, решения и важные наблюдения, чтобы через неделю не восстанавливать контекст по чатам и логам.

## Документы

- [bots-and-railway.md](bots-and-railway.md) - основная рабочая документация по Telegram- и Discord-сервисам, Railway, переменным окружения, деплою и runbook.
- [telegram-analytics-architecture.md](telegram-analytics-architecture.md) - архитектура аналитики Telegram-бота: таблицы, события, воронка, будущие отчеты.
- [current-state.md](current-state.md) - короткое актуальное состояние проекта: что работает, где данные, что требует проверки.
- [project-log.md](project-log.md) - журнал изменений, проверок и решений по датам.

## Как Вести Документацию

Когда меняем код или настройки, обновляем один из документов:

- меняется деплой, Railway, volume, env или команды запуска - обновить `bots-and-railway.md`;
- меняется логика аналитики, таблицы SQLite или события - обновить `telegram-analytics-architecture.md`;
- появилась важная проверка, инцидент или решение - добавить запись в `project-log.md`;
- изменилось общее состояние проекта - обновить `current-state.md`.

Правило безопасности: не писать в документы реальные токены, полные номера телефонов, приватные Railway variables и персональные данные пользователей. Для проверок используем агрегаты или маскированные значения.
